# 공공직군 행정업무 슈퍼앱 PRD

## 1. 개요

공공직군 행정업무 슈퍼앱은 팀 일정 공유, 엑셀 업무 자동화, 민원 대응 지원, 행정 뉴스 수집을 하나의 웹 애플리케이션에서 제공하는 업무 지원 도구이다. 반복적인 행정 업무를 줄이고, 팀 단위 정보 공유와 민원 응대 품질을 높이는 것을 목표로 한다.

## 2. 목표

- 팀원의 휴가, 근무, 출장 일정을 한눈에 확인한다.
- 엑셀 파일을 특정 컬럼 기준으로 분할하거나 여러 파일을 병합한다.
- 첨부된 민원 매뉴얼을 기반으로 대응 방식과 답변 스크립트를 생성한다.
- 공공 행정 관련 뉴스를 매일 아침 자동 수집해 업무 시작 전에 확인할 수 있게 한다.

## 3. 사용자

- 행정 업무 담당자
- 팀장 및 일정 관리 담당자
- 민원 응대 담당자
- 정책, 홍보, 기획 관련 뉴스 모니터링 담당자

## 4. 핵심 기능

### 4.1 팀원 스케쥴 관리

#### 기능 설명

팀원들의 휴가, 근무, 출장, 교육, 외근 등 일정을 공유하는 캘린더 기능이다.

#### 주요 요구사항

- 월간, 주간, 일간 캘린더 뷰를 제공한다.
- 일정 유형을 휴가, 근무, 출장, 교육, 기타로 구분한다.
- 팀원별, 일정 유형별 필터를 제공한다.
- 일정 등록, 수정, 삭제 기능을 제공한다.
- 일정에는 제목, 담당자, 시작일시, 종료일시, 유형, 장소, 메모를 포함한다.
- 같은 시간대 일정 충돌 여부를 표시한다.

#### 성공 기준

- 사용자는 팀 전체 일정을 5초 이내에 확인할 수 있다.
- 사용자는 필터를 통해 특정 팀원 또는 특정 유형의 일정만 볼 수 있다.

### 4.2 엑셀 업무 자동화

#### 기능 설명

행정 업무에서 자주 발생하는 엑셀 분리, 병합 작업을 자동화한다.

#### 주요 요구사항

- `.xlsx`, `.xls`, `.csv` 파일 업로드를 지원한다.
- 특정 컬럼 값을 기준으로 엑셀 파일을 여러 개로 분할한다.
- 여러 엑셀 파일을 하나의 파일로 병합한다.
- 병합 시 컬럼명이 다른 경우 사용자에게 매핑 또는 오류를 안내한다.
- 처리 결과를 다운로드 가능한 파일로 제공한다.
- 업로드 파일은 처리 완료 후 자동 삭제하는 정책을 적용한다.

#### 성공 기준

- 사용자는 컬럼 기준 분할 작업을 수동 필터링 없이 완료할 수 있다.
- 10MB 이하 파일은 일반 업무 환경에서 30초 이내 처리하는 것을 목표로 한다.

### 4.3 민원 대응 챗봇

#### 기능 설명

첨부된 민원 매뉴얼을 기반으로 민원 대응 방식, 유의사항, 답변 스크립트 초안을 생성하는 챗봇이다.

#### 주요 요구사항

- PDF, DOCX, TXT 등 민원 매뉴얼 업로드를 지원하는 구조로 설계한다.
- 사용자가 민원 내용을 입력하면 관련 매뉴얼 근거를 검색한다.
- 답변은 대응 요약, 권장 스크립트, 확인 필요 사항으로 구분해 제공한다.
- 민감정보 입력 금지 안내를 제공한다.
- 챗봇 답변은 최종 공식 답변이 아니라 담당자 검토용 초안임을 표시한다.
- 향후 RAG 기반 검색과 LLM 연동이 가능하도록 모듈을 분리한다.

#### 성공 기준

- 사용자는 민원 내용 입력 후 답변 초안을 받을 수 있다.
- 답변에는 담당자가 검토해야 할 추가 확인 사항이 포함된다.

### 4.4 뉴스 기사 수집

#### 기능 설명

공공 행정, 지방자치, 정책, 민원, 복지, 안전 등 관련 뉴스를 매일 아침 자동 수집한다.

#### 주요 요구사항

- 지정 키워드 기반 뉴스 검색을 수행한다.
- 매일 아침 정해진 시간에 수집되도록 스케줄러 구조를 둔다.
- 제목, 출처, 게시일, 링크, 요약을 저장한다.
- 중복 기사는 URL 또는 제목 기준으로 제거한다.
- 사용자는 수집된 뉴스를 날짜별, 키워드별로 조회할 수 있다.

#### 성공 기준

- 사용자는 매일 아침 최신 행정 관련 뉴스를 한 화면에서 확인할 수 있다.
- 중복 기사가 반복 노출되지 않는다.

## 5. 비기능 요구사항

- 프론트엔드는 TypeScript, Vite, React를 사용한다.
- 백엔드는 Python, FastAPI를 사용한다.
- Python 의존성 및 가상환경은 반드시 `uv`로 관리한다.
- 데이터베이스는 SQLite를 사용한다.
- 초기 버전은 단일 PC 또는 내부망 배포를 우선한다.
- 민원 매뉴얼과 업로드 파일에는 개인정보가 포함될 수 있으므로 저장 기간과 접근 권한을 제한한다.

## 6. MVP 범위

- 팀 일정 CRUD 및 캘린더 조회
- 엑셀 파일 분할 및 병합
- 매뉴얼 텍스트 기반 민원 답변 초안 생성 구조
- 키워드 기반 뉴스 수집 및 목록 조회
- SQLite 기반 로컬 데이터 저장

## 7. 제외 범위

- 기관 내부 인증 시스템 연동
- 전자결재 시스템 연동
- 대규모 다중 기관 배포
- 법적 효력을 갖는 자동 민원 답변 발송
- 실시간 메신저 또는 그룹웨어 연동

## 8. 주요 리스크

- 민원 챗봇 답변이 공식 답변처럼 오인될 수 있다.
- 뉴스 수집은 언론사 또는 포털 정책에 따라 접근 방식이 제한될 수 있다.
- 엑셀 파일 형식이 다양해 예외 처리가 많이 필요할 수 있다.
- SQLite는 초기 MVP에는 적합하지만 동시 사용자가 많아지면 DB 전환이 필요할 수 있다.

## 9. 향후 확장

- 권한 기반 사용자 관리
- Outlook, Google Calendar 등 외부 캘린더 연동
- 기관별 민원 매뉴얼 버전 관리
- 뉴스 키워드 추천 및 자동 분류
- PostgreSQL 전환
- Docker 기반 배포
