# 공공직군 행정업무 슈퍼앱 기능 요구서

## 1. 개요

공공직군 행정업무 슈퍼앱은 팀원 일정 관리, 엑셀 업무 자동화, 민원 대응 초안 작성, 공공행정 뉴스 수집을 한 화면에서 처리하기 위한 내부 업무 도구다.

이 문서는 MVP 기준의 기능 범위, 사용 시나리오, 성공 기준을 정의한다.

## 2. 목표

- 팀원들의 휴가, 근무, 출장 일정을 한 곳에서 확인하고 관리한다.
- 반복적인 엑셀 분할/병합 작업을 자동화한다.
- 민원 매뉴얼을 바탕으로 상담 스크립트와 대응 체크리스트 초안을 만든다.
- 공공행정 관련 뉴스를 매일 수집하고 이력을 보관한다.

## 3. 사용자

- 행정 담당자
- 팀 일정 운영자
- 민원 응대 담당자
- 정책/행정 뉴스 모니터링 담당자

## 4. 기능 요구사항

### 4.1 팀원 스케줄 관리

- 구성원을 등록, 수정, 비활성화할 수 있어야 한다.
- 일정은 구성원과 연결하거나 미배정 상태로 저장할 수 있어야 한다.
- 일정 유형은 휴가, 근무, 출장, 일반 등으로 구분할 수 있어야 한다.
- 주간 리스트와 월간 캘린더에서 일정을 확인할 수 있어야 한다.
- 구성원, 직책, 연락처, 일정 제목으로 검색할 수 있어야 한다.

### 4.2 엑셀 업무 자동화

- 업로드된 파일명을 기준으로 특정 컬럼 값을 읽어 분할할 수 있어야 한다.
- 여러 파일을 병합해 하나의 결과 파일을 생성할 수 있어야 한다.
- 결과 파일은 다운로드 가능한 형태로 저장되어야 한다.
- 입력 파일은 `backend/app/storage/uploads` 디렉터리에서 찾는다.

### 4.3 민원 대응 챗봇

- 민원 매뉴얼을 저장하고 목록으로 확인할 수 있어야 한다.
- PDF, DOCX, TXT, MD, CSV, JSON, LOG 형식의 매뉴얼을 업로드할 수 있어야 한다.
- 사용자가 질문을 입력하면 관련 매뉴얼을 참고한 답변 초안을 생성해야 한다.
- 응답은 요약, 추천 스크립트, 체크리스트, 주의 문구를 포함해야 한다.
- 생성된 대화는 DB에 기록되어야 한다.

### 4.4 뉴스 기사 수집

- 공공행정 관련 뉴스를 날짜 기준으로 조회할 수 있어야 한다.
- 지정한 날짜에 대해 수동 수집을 수행할 수 있어야 한다.
- 매일 아침 전일 뉴스가 자동으로 수집되어야 한다.
- 크롤링 실행 이력을 확인할 수 있어야 한다.

## 5. 비기능 요구사항

- 모든 데이터는 UTF-8 기준으로 저장한다.
- 로컬 MVP는 SQLite를 사용한다.
- 기능별 UI는 한글 우선으로 제공한다.
- 백엔드와 프론트엔드는 분리된 구조를 유지한다.
- 오류 발생 시 사용자가 원인을 파악할 수 있는 메시지를 제공한다.

## 6. 성공 기준

- 구성원과 일정 CRUD가 정상 동작한다.
- 주간/월간 뷰에서 일정 확인이 가능하다.
- 엑셀 분할/병합 작업이 실제 결과 파일을 생성한다.
- 민원 챗봇이 매뉴얼 기반 초안을 생성한다.
- 뉴스 수집 및 실행 이력이 정상 기록된다.
- FE-BE, BE-DB 상태 확인 화면이 정상 응답한다.

## 7. 제외 범위

- SSO 또는 사내 계정 연동
- 실시간 협업 편집
- 고급 권한 관리
- 상용 LLM 연동
- 배포형 대규모 DB 이관
